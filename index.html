Deploy Real-Time Attendance System on AWS EC2 Instance

Follow this step-by-step guide to deploy your Streamlit-webrtc app in AWS EC2.

1. Setting up Code for Deployment

  1. Create a directory: mkdir attendance_system_app
  2. Navigate to the folder: cd attendance_system_app
  3. Create a virtual environment: python -m venv virtualenv
  4. Activate the virtual environment:
    • For Windows (Git Bash): source virtualenv/Scripts/activate
    • For Linux/Mac: source virtualenv/bin/activate
  5. Create an app directory and copy your application code.
  6. Install required packages: pip install -r requirements.txt
  7. Run the application: streamlit run Home.py

2. Configure Streamlit-webrtc for Cloud

  • Update your code to configure the STUN server.
  • Create a bash file configure.sh with Apache2 virtual host configuration.
  • Create and run main.sh to automate the deployment.

3. Push Code to GitHub

  1. Create a new GitHub repository.
  2. Push your local code using Git commands.

4. Deploy on AWS EC2

  1. Create an EC2 instance with Ubuntu OS.
  2. Connect to the instance and update the system.
  3. Install and configure Apache2.
  4. Clone your repository and install required packages.
  5. Run the Streamlit app and configure inbound rules for HTTP/HTTPS.

5. Configure HTTPS with Apache2

  1. Run the configure.sh file to set up HTTPS.
  2. Restart the Apache2 server.
  3. Update inbound rules for HTTPS traffic.

6. Use PM2 for Job Management

  • Install pm2 using npm: npm install pm2 -g
  • Start the app: pm2 start "streamlit run Home.py" --name "Attendance App"
  • Ensure continuous execution with pm2 startup.

7. Configure STUN/TURN Server

  • Install and configure coturn.
  • Set up ports and update inbound rules for the TURN server.
